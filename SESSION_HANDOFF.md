## 1. What this is

A **Windows desktop app (Electron, portable `.exe`)** called **Route Template Mapper**, built from scratch this session. It:
- Reads a **SYNAOS map JSON**, finds its **navigation graphs**, and pairs each with a **route-template bundle** (templates + mappings).
- Matched graphs auto-load a known bundle; unmatched graphs let you start from a template style / blank / "no route template".
- Has a **full visual map editor** (nodes, edges, stations, groups, chargers, traffic lights, areas, per-graph translation, background PNG, grid/rulers).
- **Exports** one route-template file per graph **and** the full map, into one chosen folder.

It extends two reference apps that live in the working dir (read-only references, do not ship):
- `Robot Template Editor/` — the original Electron + vanilla-JS route-template editor (the UX/JSON base).
- `MapGenerator/` — the SYNAOS Python/PySimpleGUI map editor (the map-editor + coordinate-transform reference).

## 4. Data model (both JSON shapes)

**Map JSON** (SYNAOS): `{ id, name, description, navigationGraphs[], waitingSpots[], handlingStationGroups[], chargingStations[], trafficLights[], areas[], emergencyDetectors[], limitedCapacityAreas[] }`
- `navigationGraph = { id, agvTypes[{vendorName,typeName}], graphToMapTransformation:{xTranslation,yTranslation,zTranslation,zRotation}, nodes[{id,graphX,graphY,graphZ,[parkingSpot:{allowLoadedAgvs}],[emergencySpot]}], edges[{id,startNodeId,endNodeId}] }`
- `handlingStationGroup = { id, controlled, justInSequence, stationAutomaticSelectionMode(ALWAYS/ONLY_PICK/ONLY_DROP/NEVER), handlingStations[{id,mapX,mapY,mapZ,width,height,length,actions,accessNodes[{navigationGraphId,nodeId}]}], waitingSpots }`
- charger `{id,mapX,mapY,mapZ,length,width,height,controlled,accessNodes[]}` · light `{id,mapX,mapY,mapZ,[scenarioIndicatorCounter]}` · area `{id,points[{mapX,mapY}]}` · emergencyDetector `{id,areas:[areaName]}` · limitedCapacityArea `{id,capacity,nodes[{nodeId,navigationGraphId}]}`

**Route-template bundle:** `{ templates[{id,nodes?,edges?,steps[{reference,type:'node'|'edge',actions[{actionName,blockingType,actionParameters[{key,value,dataType}],targetHandlingStationCondition:{externalId,loaded}|null}],attributes[{key,value}]}]}], mappings[{id,template,actionMapping,conditions|null,entries[{key,value}],navigationGraphId}] }`
- Placeholder tokens (e.g. `DROP_CONTAINERX_A`) live in template values; mappings fill them.
- **Gotcha:** Tusk-style templates declare NO top-level `nodes[]`/`edges[]` — node refs (A, B) come only from `steps` of `type:'node'` (`templateNodeRefs`).
- Known graphs baked in: **`KMP 400P-1-5G diffDrive`** (kuka) + **`tusk`**, each paired 1:1 with a provided bundle.

## 5. CRITICAL correctness detail — coordinate transform

The transform is **`map = Rot(zRotation)·(graphLocal − translation)`** (SUBTRACT the translation), matching SYNAOS `MapComponents.transformCoordinates`. Inverse (map→local) = `Rot(−rz)·map + translation`. In `dist/app.js` these are `g2m` / `m2g`. Getting the sign wrong (add) makes graphs land in the wrong place — they must OVERLAP in the shared map frame (Tusk node 0003 → map ≈ (26.12, 10.30), next to its station T1). **Nodes are stored in local `graphX/graphY`; export writes those unchanged. Only on-screen placement uses the transform.**

## 6. Architecture notes (dist/app.js)

- `state.view`: `'landing' | 'route' | 'map'`. `render()` routes on it.
- Route side: `state.graphs[]` each `{ graphId, agvTypes, nodeIds, edgeIds, matched, baseChosen, noRouteTemplate, bundle }`. `state.bundle` is a pointer to the active bundle (`syncActiveBundle`). Matching by graph id + exact sorted node/edge id sets (`findKnownGraph`). "Generate from map" (`generateFromMap`) makes edge-mappings per edge + pickup/dropoff per handling station.
- `state.mapEd` (map editor): `gi, mode(select/node/edge/station/charger/light/area), selset[], grid, coords, translatePanel, marquee, view{x,y,zoom}, decimals, edgeBidir, edgeFrom, areaDraft, ungroupedStations, bg, drag, dialog, _space, _fitted, _dirty`.
  - Canvas is `#map-svg`; screen coords via `w2s`/`s2w`; `paintCanvas()` rebuilds `svg.innerHTML` (screen-space). Interaction listeners are document-level in `init()`.
  - **Left-click = select** (Shift+click add, Shift+drag = marquee `marqueeSelect`, drag = move all selected, Delete = delete all). **Right-click (`onContextMenu`) = create** the active tool's type.
  - **Access nodes:** only while the matching tool is active (`ed.mode === sel.type`) does clicking a node toggle it in the element's `accessNodes` (`toggleAccessNode` + `accessTarget`, works for **stations AND chargers**); green dashed link `.m-acc`/`.m-accnode`. In other tools a node click just selects it.
  - Grid+rulers (`mapGridInner`/`mapRulerInner`, `niceStep`), live readout `#map-readout` + crosshair `#map-xh-*`, live translate panel (`translatePanelHTML`, `trans` action on `input`), `coords` toggle (loc+map labels on active-graph nodes).
  - Stations live in `handlingStationGroups[].handlingStations` PLUS editor-only `state.mapEd.ungroupedStations`; must be grouped to export.
  - `canonicalizeMap()` rounds to `mapEd.decimals` and matches the sample field order.
  - Reconcile (`reconcileGraphsAfterMapEdit`) on returning to route view: keep existing per-graph bundles by id, add new graphs (matched→known, else base chooser), drop removed — non-destructive.
- Icons (Scheme A, CSS in styles.css + shapes in `mapCanvasInner`): node = blue circle `#2563eb`, station = teal rounded-rect `#0d9488`, charger = red hexagon polygon `#dc2626`, light = amber triangle polygon `#f59e0b`, area = violet outline `#7c3aed`. Selected = gold `#f5b301`. Ungrouped station = amber.
- **No route template:** graph flag `noRouteTemplate` (base-chooser "No route template" tile = `base:none`, or footer `graph:noTemplate` button; reverse = `graph:withTemplate`). Shows a placeholder + "no template" pill, excluded from the export picker.
- Persistence: localStorage `routeTemplateMapper.session.v1` = `{map, mapName, graphs, gi, mapEd:{decimals, ungroupedStations, bg(path/scale only, no dataUrl)}}`. Theme + user samples in their own keys. Background re-read by path via IPC on resume (`reloadBackground`).
- IPC (main.mjs): `open-map-dialog`, `open-bundle-dialog`, `save-bundle-dialog`, `export-graphs-dialog` (folder → write N files), `open-image-dialog`, `read-image`.

## 8. Open ideas / known limitations (not yet done)

- Area **points aren't individually drag-editable** (you redraw an area to change it) — could add point-level editing.
- Marquee selects active-graph nodes + all stations/chargers/lights (mixed types allowed) — could restrict to one type.
- Traffic-light `scenarioIndicatorCounter`, emergency detectors, limited-capacity areas are **pass-through / not editable in UI** (preserved on load→export).
- Background image + folder export need the **packaged exe** (Electron IPC); they no-op in a plain browser.
- "No route template" is offered in the base chooser (unmatched graphs) + a footer button; matched graphs use the footer button.
- Preview JSON panel isn't shown while a graph is in the "no route template" placeholder (edge case).
- Consider a native menu bar / more polish; larger maps (hundreds of nodes) repaint the whole SVG on pan — fine for the sample, could be optimized.
